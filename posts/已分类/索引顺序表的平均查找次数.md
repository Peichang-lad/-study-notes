# 📝 笔记：索引顺序文件 ASL (平均查找长度) 计算

## 1. 核心逻辑

**索引顺序文件查找 = 查索引表 + 组内顺序查找**

因此：**总平均查找次数 (ASL) = 查索引平均次数 + 组内查找平均次数**

**基本假设（参数定义）：**

- **$N$**：文件总记录数
    
- **$m$**：每组的记录数
    
- 索引表总项数（组数）：**$\frac{N}{m}$**（假设整除）
    

## 2. 常用推导模型分类

### 模型 A：索引表采用「顺序查找」（408 默认情况）

如果题目没有特别说明，默认索引表和组内都使用顺序查找。

- **步骤 1：查索引表**
    
    索引表共有 $\frac{N}{m}$ 项，顺序查找的平均次数为一半：
    
    $$平均次数 = \frac{\frac{N}{m}}{2} = \frac{N}{2m}$$
    
- **步骤 2：组内查找**
    
    进入某组后，该组有 $m$ 条记录，顺序查找的平均次数为：
    
    $$平均次数 = \frac{m+1}{2}$$
    
- **推导公式：**
    
    $$ASL = \frac{N}{2m} + \frac{m+1}{2}$$
    

> **🌰 举个例子：**
> 
> 假设文件总记录数 $N = 10000$，每组记录数 $m = 100$。
> 
> - 索引项数 = $10000 / 100 = 100$ 项
>     
> - 查索引 = $100 / 2 = 50$ 次
>     
> - 组内查找 = $(100 + 1) / 2 = 50.5$ 次
>     
> - **总 $ASL = 50 + 50.5 = 100.5$ 次**
>     

### 模型 B：索引表采用「二分查找」

如果题目明确说明索引表是有序的，且采用二分（折半）查找。

- **步骤 1：查索引表**
    
    对 $\frac{N}{m}$ 项进行二分查找：
    
    $$平均次数 \approx \log_2(\frac{N}{m})$$
    
- **步骤 2：组内查找**
    
    组内仍为无序的顺序查找：
    
    $$平均次数 = \frac{m+1}{2}$$
    
- **推导公式：**
    
    $$ASL = \log_2(\frac{N}{m}) + \frac{m+1}{2}$$
    

## 💡 408 考场做题判断技巧

在做 408 统考题时，严格按照题干给定的条件选择公式：

1. **题目没说索引表怎么查** $\rightarrow$ **默认顺序查索引**（用模型 A：$ASL = \frac{N}{2m} + \frac{m+1}{2}$）
    
2. **题目明确说“索引表有序”或“采用二分查找”** $\rightarrow$ **用二分**（用模型 B：$ASL = \log_2(\frac{N}{m}) + \frac{m+1}{2}$）
