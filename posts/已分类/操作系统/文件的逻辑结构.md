> **索引顺序文件（Indexed Sequential File，ISF）**是文件逻辑结构里面一个比较重要的结构，它可以理解成：

**索引顺序文件（Indexed Sequential File，ISF）**是文件逻辑结构里面一个比较重要的结构，它可以理解成：

> **顺序文件 + 索引表**

它是为了弥补顺序文件“查找慢”的问题。

---

## 1. 先看顺序文件的问题

假设学生文件：

```id="9xqz8u"
记录：

001 张三
002 李四
003 王五
...
1000 赵六
```

按照学号排序。

如果找：

```
001
```

很快。

但是找：

```
900
```

顺序文件：

```id="x7j9t1"
001
002
003
...
900
```

可能要一个个扫。

时间复杂度接近：

```
O(n)
```

---

# 2. 索引顺序文件的思想

给它加一个索引：

例如：

原文件：

```id="a5gk3r"
001 张三
002 李四
003 王五
...
1000 赵六
```

建立：

```
索引表：

第1组 001~100 → 地址A
第2组 101~200 → 地址B
第3组 201~300 → 地址C
...
```

结构：

```
索引表
 |
 |
 ↓

数据文件

001
002
003
...
100
101
102
...
```

---

# 3. 查找过程

比如找：

```
250号学生
```

### 第一步：

查索引：

```
201~300 → 地址C
```

直接定位到第三组。

### 第二步：

在这一组里面顺序查：

```
201
202
...
250
```

---

所以：

不是全文件查。

而是：

```
索引定位
    ↓
局部顺序查找
```

---

# 4. 和索引文件区别（重点）

很多人混：

## 索引文件

相当于：

每条记录都有目录：

```
001 → 地址1
002 → 地址2
003 → 地址3
```

查找直接跳。

速度快。

但是：

索引表很大。

---

## 索引顺序文件

不是每条都有索引。

是一组一组：

```
索引：

001组 → 地址A
101组 → 地址B
201组 → 地址C
```

组内再顺序找。

---

# 5. 一个生活类比

顺序文件：

看书：

```
一页一页翻
```

索引顺序文件：

书后面：

```
目录：

操作系统 → 第50页
网络 → 第100页

然后翻附近
```

索引文件：

像搜索引擎：

输入关键词：

直接跳到答案。

---

# 408记忆：

> **索引顺序文件是顺序文件的改进形式，通过建立索引表，将文件分成若干组，先通过索引定位记录所在组，再在组内顺序查找。**

它主要考：

- 三种文件结构比较
    
- 查找效率
    
- 为什么比顺序文件快
    

这个和后面的**索引分配（文件物理结构）**名字很像，但不是一个东西，注意区分。

|**文件类型**|**组织方式**|**查找速度**|**索引大小**|**空间占用**|
|---|---|---|---|---|
|**顺序文件**|一个接一个|慢|无|少|
|**索引顺序文件**|分组+索引|较快|较小|中等|
|**索引文件**|全索引|最快|大|多|
