## 1. 什么是中断？

一句话：

> CPU在执行程序时，遇到外部/内部事件，暂停当前程序，转去处理事件，处理完回来继续。

过程：

```
主程序
  |
  | 运行
  ↓
中断请求
  ↓
保存现场
  ↓
执行中断服务程序
  ↓
恢复现场
  ↓
返回主程序
```

---

# 2. 中断响应 vs 中断处理

很多人混。

## 中断响应（硬件完成）

CPU发现中断后自动做：

① 关中断  
② 保存断点   
③转入中断服务程序

## 中断处理（软件完成）

中断服务程序做：

① 保护现场（有些教材放这里）  
② 执行服务程序  
③ 恢复现场  
④ 开中断  
⑤ 中断返回

考试看教材，有的划分不同，但核心一样。

---

# 3. 为什么要保存现场？

现场 = CPU当前状态

包括：

- PC（程序计数器）
    
- PSW（程序状态字）
    
- 通用寄存器
    
- 标志寄存器
    

比如：

主程序：

```
R1=10
R2=20
```

进入中断：

```
中断程序修改R1
```

回来：

如果没保存：

```
R1变了
主程序崩
```

所以：

```
保存 → 修改 → 恢复
```

---

# 4. 保存现场在哪里？

两个地方：

## 硬件保存

一般保存：

- PC
    
- PSW
    

因为CPU必须知道：

“回来去哪”

---

## 软件保存

一般保存：

- 通用寄存器
    

因为数量多，硬件做成本高。

---

# 5. 开中断和关中断

重点！

## 开中断

允许新的中断进来：

```
中断A执行
    |
    中断B来了
    |
    A暂停
    |
    执行B
```

这就是：

### 中断嵌套

---

## 关中断

禁止普通中断：

```
中断A执行

关中断

中断B来了

等待
```

---

# 6. 多重中断怎么判断能不能插入？

看优先级：

假设：

```
A 优先级高
B 优先级低
```

A执行：

来了B：

```
不能打断
```

B执行：

来了A：

```
可以打断
```

口诀：

> 高优先级可以打断低优先级，反过来不行。

# 7. 中断屏蔽字（很爱考）

每个中断有一个屏蔽位：

例如：

```
中断屏蔽字：

键盘  1
磁盘  0
时钟  0
```

1表示：

屏蔽它。

所以：

```
键盘来了 ×
磁盘来了 √
时钟来了 √
```

---

# 8. 屏蔽和优先级区别

这个特别容易错。

优先级：

决定：

> 谁先处理

屏蔽：

决定：

> 让不让它进

例如：

磁盘优先级最高：

但是：

屏蔽位=1

那么：

也不能进。

---

# 9. 不可屏蔽中断 NMI

你刚问的。

特点：

- 优先级最高
    
- 不受关中断影响
    
- 不能被软件屏蔽
    

例如：

- 电源故障
    
- 硬件故障

# 10. 中断向量

CPU怎么知道跳哪里？

靠：

### 中断向量

类似地址表：

```
中断号
 ↓
地址

键盘 → 1000H
磁盘 → 2000H
```

CPU查表：

跳过去。

# 11. 中断和异常区别

考研常考：

||中断|异常|
|---|---|---|
|来源|外部|内部|
|异步/同步|异步|同步|
|例子|键盘|除零|

例如：

程序：

```
1÷0
```

不是外部来了，而是程序自己产生：

异常。

---

# 12. DMA和中断区别（超级高频）

DMA：

直接内存访问

传统：

```
外设
 ↓
CPU
 ↓
内存
```

DMA：

```
外设
 ↓
内存
```

CPU只负责开始和结束。

---

# 13. 中断为什么不能一直开着？

因为：

如果中断无限嵌套：

```
A
 ↓
B
 ↓
C
 ↓
D
```

栈爆炸。

所以：

进入关键区域：

```
关中断
```

---

最后给你一个总口诀：

> **响应靠硬件，处理靠软件；  
> 开中断能嵌套，关中断防打扰；  
> 高级打低级，低级等高级；  
> 屏蔽决定进不进，优先决定谁先行；  
> NMI最高级，关中断也得听。**

这部分基本覆盖考研计组中断章节的大部分考点。你前面问的那个“400ns占用率”其实就是中断和CPU时间配合的延伸。
