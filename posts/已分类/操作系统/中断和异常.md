中断（Interrupt）和异常（Exception）是计算机系统中极其重要的概念，它们是操作系统与硬件交互、管理控制流的核心机制。简单来说，它们都是打断 CPU 当前执行程序的正常流程，转而去处理特定事件的机制。

虽然这两个词在日常中有时会被混用（有些教材统称为“广义上的中断”），但在计算机体系结构中，它们有严格的区分。核心区别在于**事件的来源**和**发生的时间**。

## 一、 中断（Interrupt / 异步中断 / 外部中断）

中断是由 CPU **外部**的硬件设备触发的电信号，目的是通知 CPU 某个外部事件已经发生，需要 CPU 停下手头的工作来处理。

- **异步性（Asynchronous）：** 中断可以在任何时间发生，与 CPU 当前正在执行的指令没有任何关系。它是不可预测的。
    
- **来源：** 外部 I/O 设备（如键盘按下、鼠标移动、磁盘读写完成、网卡接收到数据包）、硬件定时器（产生时钟中断，用于操作系统的进程调度）等。
    
- **处理机制：** CPU 会在当前正在执行的**指令结束后的指令周期末尾**，检查是否有中断请求到达。如果有，再去响应中断。
    

## 二、 异常（Exception / 同步中断 / 内部中断）

异常是由 CPU **内部**在执行当前指令时产生的，通常是因为遇到了某种错误条件或执行了特殊的指令。

- **同步性（Synchronous）：** 异常与 CPU 当前执行的特定指令直接绑定。如果重新执行同一段程序，在相同的机器状态下，异常一定会在同一个地方再次发生。
    
- **来源：** 算术溢出、除零错误、非法的内存访问请求（越界）、缺页、系统调用指令等。
    
- **分类（通常分为三种）：**
    
    1. **故障（Fault）：** 可恢复的错误。例如**缺页异常（Page Fault）**。当指令引用了一个不在物理内存中的虚拟地址时触发。操作系统会将所需页面调入内存，然后**重新执行**触发异常的那条指令。
        
    2. **陷阱（Trap / 陷入）：** 有意为之的异常。最典型的应用就是**系统调用（System Call）**。用户态程序通过执行特定的陷阱指令（如 x86 的 `int 0x80` 或 `syscall`）主动切换到内核态，请求操作系统提供特权服务（如读写文件）。处理完成后，返回并执行陷阱指令的**下一条**指令。
        
    3. **终止（Abort）：** 严重的、不可恢复的硬件错误。例如内存奇偶校验错误或硬件总线故障。此时程序无法继续执行，操作系统通常会直接中止整个系统或杀死相关进程，**不会返回**。
        

## 三、 核心区别对比

为了更直观地理解，我们可以通过以下表格进行对比：

|**特性**|**中断 (Interrupt)**|**异常 (Exception)**|
|---|---|---|
|**触发来源**|外部硬件（I/O 设备、定时器等）|内部 CPU（执行当前指令时产生）|
|**同步性**|异步（与当前指令序列无关）|同步（由当前特定指令触发）|
|**可预见性**|随机、不可预测|确定、可预测（相同条件下必然触发）|
|**返回地址**|被打断指令的**下一条**指令|故障返回**当前**指令；陷阱返回**下一条**指令；终止**不返回**|
|**主要目的**|提高外设利用率，实现系统并发与多任务|处理程序运行错误，实现内核态/用户态的安全切换|

## 四、 统一的处理流程

尽管触发原因不同，但现代 CPU 处理中断和异常的基本硬件流程是高度相似的：

1. **保存现场：** CPU 硬件自动将当前的程序计数器（PC / EIP）、程序状态字（PSW / EFLAGS）等关键寄存器状态压入内核栈中。
    
2. **查找向量表：** 根据中断或异常的特定编号（Interrupt Vector），在中断向量表（IDT, Interrupt Descriptor Table）中查找对应的处理程序入口地址。
    
3. **执行处理程序：** 将 CPU 权限切换至内核态（如果之前在用户态），并跳转到操作系统预先写好的中断服务例程（ISR）或异常处理程序中执行。
    
4. **恢复现场并返回：** 处理完成后，执行特定的中断返回指令（如 x86 的 `iret`），从内核栈中弹出并恢复之前保存的寄存器状态，CPU 带着恢复的现场继续执行原程序。
