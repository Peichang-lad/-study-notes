> 循环队列中 front==rear 到底是空还是满？牺牲一个单元法的判空、判满条件详解。


在数据结构中，循环队列（通常基于数组实现）为了区分“队空”和“队满”的物理状态，最经典且最常用的方法是**牺牲一个存储单元**。

假设队列的数组容量为 `MaxSize`，队头指针 `front` 指向队列的第一个元素，队尾指针 `rear` 指向队列最后一个元素的下一个位置。

### 1. 判断为空的条件

- **条件公式：** `front == rear`
    
- **原理解释：** 无论是刚初始化的队列，还是经过一系列入队、出队操作后元素被全部清空的队列，只要队头指针和队尾指针指向数组中的同一个位置，就说明队列中没有任何有效元素。
    

### 2. 判断为满的条件

- **条件公式：** `(rear + 1) % MaxSize == front`
    
- **原理解释：** 如果不牺牲一个空间，当队列真正装满时，`rear` 转了一圈又会追上 `front`，导致 `front == rear`。这就和“判空”的条件起冲突了。因此，我们**规定当队尾指针的下一个位置就是队头指针时，队列算作满**。这里的 `% MaxSize` 是取模（求余）运算，用于实现数组下标的“循环”效果。
    

---

### 补充：如果不牺牲存储单元的判断方法

如果你希望 100% 利用数组的所有空间（不空出那个位置），可以通过在结构体中增加额外的辅助变量来实现：

**方法 A：增加一个 `size` 变量**

- 初始化时 `size = 0`。每次入队 `size++`，每次出队 `size--`。
    
- **判空条件：** `size == 0`
    
- **判满条件：** `size == MaxSize`
    

**方法 B：增加一个 `tag` 标志位**

- 初始化时 `tag = 0`。因为引起 `front == rear` 只可能是两种操作：入队导致满，或者出队导致空。
    
- 每次成功入队后，将 `tag` 设为 `1`；每次成功出队后，将 `tag` 设为 `0`。
    
- **判空条件：** `front == rear && tag == 0` （指针重合，且最后一次操作是出队，说明全出光了）
    
- **判满条件：** `front == rear && tag == 1` （指针重合，且最后一次操作是入队，说明全装满了）
究竟是rear追上了front还是front追上了rear
