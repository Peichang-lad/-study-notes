## 图论八大核心算法总览

### 一、 图的遍历 (Traversal)

这是所有高级图算法的基础，核心在于“不重不漏”地访问所有顶点。

- **深度优先搜索 (DFS)**
    
    - **核心思想**：一条路走到黑，不撞南墙不回头（回溯）。类似树的**先序遍历**。
        
    - **实现基础**：依赖**栈**（通常通过递归实现）。
        
    - **常考应用**：判断图是否有环、求连通分量、生成拓扑序列（逆后序）。
        
- **广度优先搜索 (BFS)**
    
    - **核心思想**：像水波纹一样层层扩散。类似树的**层序遍历**。
        
    - **实现基础**：依赖**队列**。
        
    - **常考应用**：求**无权图**的单源最短路径（因为它是按层级推进的，先找到的路径一定边数最少）。
        

### 二、 最小生成树 (Minimum Spanning Tree, MST)

解决“如何用最小的代价把所有顶点连通”的问题（无向网）。

| **对比维度**  | **Prim (普里姆) 算法**                 | **Kruskal (克鲁斯卡尔) 算法**        |     |     |     |     |     |
| --------- | --------------------------------- | ----------------------------- | --- | --- | --- | --- | --- |
| **核心策略**  | **加点法**（从单个顶点出发，每次挑离当前生成树最近的顶点加入） | **加边法**（全局挑选权值最小的边，只要不构成环就加入） |     |     |     |     |     |
| **适用场景**  | 边多点少的**稠密图**                      | 边少点多的**稀疏图**                  |     |     |     |     |     |
| **时间复杂度** |                                   |                               |     |     |     |     |     |
| **考试陷阱**  | 每次加入的必定是直接相连的最短边。                 | 选边时必须判断是否成环（通常隐含考察并查集的思想）。    |     |     |     |     |     |

### 三、 最短路径 (Shortest Path)

解决“从一个点到另一个点代价最小的路线”问题（有向/无向网）。

| **对比维度**  | **Dijkstra (迪杰斯特拉) 算法**       | **Floyd (弗洛伊德) 算法**         |     |     |     |     |
| --------- | ----------------------------- | --------------------------- | --- | --- | --- | --- |
| **解决问题**  | **单源**最短路径（求某一顶点到其他所有顶点的最短路径） | **多源**最短路径（求任意两个顶点之间的最短路径）  |     |     |     |     |
| **核心思想**  | 贪心算法（按路径长度递增的次序产生最短路径）        | 动态规划（不断尝试将各个顶点作为中转站）        |     |     |     |     |
| **时间复杂度** |                               |                             |     |     |     |     |
| **致命限制**  | **绝对不能处理带有负权边的图！**            | 能处理负权边，但**不能处理带有“负权回路”的图**。 |     |     |     |     |

### 四、 有向无环图 (DAG) 的应用

专门针对带有“方向”且“没有循环依赖”的复杂工程调度问题。

- **拓扑排序 (Topological Sort)**
    
    - **操作对象**：**AOV 网**（顶点表示活动，边表示优先关系）。
        
    - **核心动作**：不断寻找入度为 $0$ 的顶点并删除（Kahn 算法）。
        
    - **核心结论**：能产生包含所有顶点的拓扑序列 $\iff$ 图中无环。
        
- **关键路径 (Critical Path)**
    
    - **操作对象**：**AOE 网**（顶点表示事件，边表示活动，边上的权值表示活动耗时）。
        
    - **核心动作**：计算事件的最早/最晚发生时间，活动的最早/最晚开始时间，从而找出**时间余量为 $0$ 的关键活动**。
        
    - **考试陷阱**（联动你图片里的 2013 年真题）：缩短关键活动可以缩短整个工期，但**缩短到一定程度，关键活动可能会变成非关键活动**，此时工期就不会再缩短了。
