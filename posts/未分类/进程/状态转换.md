> 状态转换是操作系统里**进程从出生到结束的生命周期变化**，核心就是看：**进程什么时候能运行，什么时候不能运行，什么时候结束。**

状态转换是操作系统里**进程从出生到结束的生命周期变化**，核心就是看：**进程什么时候能运行，什么时候不能运行，什么时候结束。**

经典五状态模型：

```
          创建
           |
           v
        新建态
           |
        接纳
           |
           v
        就绪态
           |
        调度
           |
           v
        运行态
        /   \
       /     \
      v       v
   阻塞态    终止态
      |
      |
      v
   就绪态
```

再加一个：

```
运行态 → 就绪态
```

这个非常重要。

完整图：

```
          新建
            |
            ↓
          就绪
            |
            ↓
          运行
        ↙      ↘
     阻塞       终止
       |
       ↓
     就绪
```

还有：

```
运行 → 就绪
```

---

## 1. 新建态 → 就绪态

什么意思？

一个进程刚创建出来：

比如你打开浏览器：

系统做：

- 分配 PCB
    
- 分配资源
    
- 加入就绪队列
    

但是注意：

它还不能运行，因为 CPU 可能正在给别的进程。

所以：

```
新建 → 就绪
```

表示：

> 进程已经准备好了，等 CPU。

---

## 2. 就绪态 → 运行态 ⭐⭐⭐

这个就是：

**调度**

CPU 从就绪队列选一个进程运行。

例如：

现在：

```
微信   就绪
QQ     就绪
浏览器 就绪
```

调度程序选浏览器：

```
浏览器：
就绪 → 运行
```

---

## 3. 运行态 → 阻塞态 ⭐⭐⭐

这个最容易错。

运行中的进程：

发现自己暂时干不了：

例如：

```
read文件
```

但是磁盘还没读完。

CPU：

不能干等。

于是：

```
运行 → 阻塞
```

原因：

等待某个事件。

常见：

- 等 I/O
    
- 等锁
    
- 等信号
    
- 等资源
    

比如：

程序：

```cpp
cin >> x;
```

你没输入：

程序卡住。

进程：

运行 → 阻塞

---

## 4. 阻塞态 → 就绪态 ⭐⭐⭐

刚才等待的事情完成。

比如：

磁盘：

```
读取完成
```

但是 CPU 现在可能被别人占用。

所以：

不是直接运行。

而是：

```
阻塞
 ↓
就绪
 ↓
运行
```

---

## 5. 运行态 → 就绪态 ⭐⭐⭐⭐

这个是很多人漏的。

原因：

CPU时间片到了。

比如时间片轮转：

一个进程：

```
运行 10ms
```

时间到了：

操作系统说：

换一个。

于是：

```
运行 → 就绪
```

不是阻塞！

因为它没有等待任何东西。

只是：

CPU让给别人。

---

## 6. 运行态 → 终止态

进程结束。

比如：

main函数执行完：

```cpp
return 0;
```

或者：

被系统杀死。

变：

```
运行 → 终止
```

---

## 一个生活类比

把进程想成人：

### 新建

出生登记：

```
刚出生
```

↓

### 就绪

人在教室排队：

```
等老师叫
```

↓

### 运行

老师叫到你：

```
正在答题
```

↓

两种：

### 阻塞

答题需要查资料：

```
去图书馆等资料
```

↓

资料到了：

```
回教室排队
```

（阻塞→就绪）

或者：

### 就绪

老师说：

```
时间到了，下一个
```

你回队伍：

（运行→就绪）

---

408最喜欢考的判断：

|情况|转换|
|---|---|
|时间片用完|运行→就绪|
|等待I/O|运行→阻塞|
|I/O完成|阻塞→就绪|
|被调度|就绪→运行|
|程序结束|运行→终止|

记一句：

**等资源：阻塞；等CPU：就绪。**

这个基本能解决大部分状态转换题。
