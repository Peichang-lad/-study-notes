> 文件的**物理结构（文件分配方式）**研究的是：

文件的**物理结构（文件分配方式）**研究的是：

> **文件的数据在磁盘上到底怎么存放。**

前面讲逻辑结构：

用户看到：

```
学生1
学生2
学生3
```

现在物理结构关心：

```
磁盘块：
100
101
102
...
```

怎么安排。

文件物理结构主要有：

1. **连续分配**
    
2. **链接分配**
    
3. **索引分配**
    

我们先讲第一个：**连续分配**。

---

# 一、连续分配是什么？

一句话：

> **一个文件占用磁盘上连续的一组块。**

比如：

文件：

```
a.txt
```

大小需要4个磁盘块。

磁盘：

```
块号：

0  1  2  3  4  5  6  7

        ↓
      a.txt

        3 4 5 6
```

那么：

```
a.txt:
起始块=3
长度=4
```

文件控制块 FCB 只需要记录：

```
起始地址
文件长度
```

---

# 二、怎么访问？

假设：

文件：

```
开始块=100
长度=5
```

那么：

```
100 101 102 103 104
```

如果访问第3个块：

公式：

[  
块号=起始块+i  
]

所以：

第3块：

[  
100+3=103  
]

直接找到。

---

# 三、连续分配优点

## 1. 顺序访问快

因为连续：

```
100→101→102→103
```

磁盘磁头移动少。

适合：

- 视频
    
- 大文件
    

---

## 2. 随机访问快（重点）

随机访问：

比如我要第50块。

直接：

```
起始地址+50
```

不用遍历。

所以：

随机访问效率高。

---

## 3. 管理简单

FCB：

只需要：

```
起始块
长度
```

简单。

---

# 四、连续分配缺点（重点）

## 1. 外部碎片

这是最大问题。

比如：

磁盘：

```
空闲：

100 101 102

200 201

300 301 302
```

总空闲：

8块

但是我要创建：

```
需要5个连续块
```

结果：

放不下。

为什么？

因为：

没有连续空间。

这叫：

**外部碎片**

---

## 2. 文件扩展困难

比如：

原来：

```
a.txt

100 101 102
```

后来变大：

需要：

```
100 101 102 103 104
```

但是：

103被别人占了。

怎么办？

只能：

移动整个文件：

```
搬到200开始
```

代价大。

# 五、连续分配示意

```
磁盘

|空|空|A|A|A|A|空|B|B|空|

        ↑
        A文件连续
```

# 六、408常考比较
| 特性   | 连续分配     |
| ---- | -------- |
| 存储方式 | 连续块      |
| 随机访问 | 快        |
| 顺序访问 | 快        |
| 记录地址 | 起始块 + 长度 |
| 碎片情况 | 外部碎片     |
| 空间   | 扩展困难     |

# 七、和索引分配区别

注意名字容易混：

连续分配：

解决：

> 文件数据块怎么放？

例如：

```
文件 → 100,101,102
```

索引分配：

解决：

> 用一个索引块记录文件所有地址

例如：

```
索引块：

100
200
500
```

---

# 八、考试一句话

> 连续分配把文件存放在磁盘连续的物理块中，具有访问速度快、随机访问方便的优点，但存在外部碎片和文件扩展困难的问题。

下一个就是**链接分配**，它正好是针对连续分配“碎片和扩展困难”改进的。你按这个顺序学会比较容易串起来。
