# 五大经典同步问题笔记

## 一、生产者-消费者问题 ⭐⭐⭐⭐⭐

### 1. 问题描述

有一个**大小为N的缓冲区**：

- 生产者：生产数据 → 放入缓冲区
    
- 消费者：取数据 → 消费
    

例如：

```
生产者 ----> 缓冲区 ----> 消费者
```

---

### 2. 核心矛盾

两个问题：

① 缓冲区满了，生产者不能生产

② 缓冲区空了，消费者不能消费

③ 多个进程访问缓冲区，需要互斥

---

### 3. 信号量设计

设：

```
mutex = 1   //互斥访问缓冲区

empty = N   //空闲位置数量

full = 0    //已有产品数量
```

---

### 4. PV过程

生产者：

```c
P(empty);     //有没有空位置

P(mutex);     //进入缓冲区

生产产品

V(mutex);

V(full);      //增加产品数量
```

消费者：

```c
P(full);      //有没有产品

P(mutex);

消费产品

V(mutex);

V(empty);
```

---

### 5. 记忆

> 生产者：先问有没有空位，再进入仓库
> 
> 消费者：先问有没有货，再进入仓库

---

# 二、多生产者-多消费者问题 ⭐⭐⭐⭐

## 1. 场景

多个生产者：

```
P1
P2
P3
 |
 ↓
缓冲区
 |
 ↓
C1
C2
C3
```

---

## 2. 本质

其实还是：

> 多个进程竞争有限资源

所以：

需要：

- 数量信号量
    
- 互斥信号量
    

---

## 3. 信号量

例如缓冲区大小N：

```
mutex=1

empty=N

full=0
```

和生产者消费者一样。

---

## 4. 注意

多生产者、多消费者不一定每个生产者生产同一种东西。

例如：

生产：

```
面包
牛奶
鸡蛋
```

消费者：

```
吃面包
喝牛奶
吃鸡蛋
```

可能要增加：

```
apple=0
milk=0
bread=0
```

表示不同资源数量。

---

# 三、抽烟者问题 ⭐⭐⭐⭐

## 1. 场景

有：

- 一个供应者
    
- 三个抽烟者
    

每个烟民缺一种材料：

```
烟草
纸
胶水
```

---

例如：

烟民A：

有烟草

缺：

纸+胶水

---

## 2. 核心矛盾

供应者随机提供两种材料：

谁需要谁抽烟。

---

## 3. 信号量

三个烟民：

```
smoker1=0

smoker2=0

smoker3=0
```

供应者：

```c
随机放材料

V(smoker对应的人)
```

烟民：

```c
P(smoker)

拿材料

抽烟
```

---

## 4. 考点

这个题考：

> 如何让进程按照条件执行

不是考互斥。

---

# 四、读者-写者问题 ⭐⭐⭐⭐⭐

## 1. 场景

共享文件：

```
多个读者 ---- 文件 ---- 写者
```

---

读：

可以多个同时读

写：

必须独占

---

## 2. 核心矛盾

读：

```
读 + 读 √
```

写：

```
写 + 写 ×

读 + 写 ×
```

---

## 3. 信号量

```
rw=1

//保护文件

mutex=1

//保护readcount

readcount=0
```

---

## 4. 读者流程

第一个读者：

加锁文件

```c
P(mutex)

readcount++

if(readcount==1)
    P(rw)

V(mutex)


读


P(mutex)

readcount--

if(readcount==0)
    V(rw)

V(mutex)
```

---

## 5. 写者

```c
P(rw)

写


V(rw)
```

---

## 6. 记忆

> 读者共享，写者独占

---

# 五、哲学家进餐问题 ⭐⭐⭐⭐⭐

## 1. 场景

5个哲学家：

```
P1  P2  P3  P4  P5
 \  |   |   |  /
   五根筷子
```

每个人需要两根筷子。

---

## 2. 核心问题

如果：

每个人先拿左筷子：

```
P1拿左
P2拿左
P3拿左
...
```

结果：

全部等待右筷子

产生：

> 死锁

---

## 3. 信号量

每根筷子：

```
chopstick[i]=1
```

---

## 4. 解决方法

### 方法1：限制人数

最多4个人同时拿筷子

```c
count=4
```

保证不会死锁。

---

### 方法2：改变拿筷子顺序

奇数：

先左后右

偶数：

先右后左

---

### 方法3：一次拿两根

```c
P(chopstick[i])

P(chopstick[(i+1)%5])
```

---

# 五个问题对比表（背这个）

|问题|资源|主要矛盾|信号量|
|---|---|---|---|
|生产者消费者|缓冲区|满/空+互斥|empty/full/mutex|
|多生产多消费|缓冲区|多个进程竞争|empty/full/mutex|
|抽烟者|材料|条件同步|smoker|
|读写者|文件|读写冲突|rw|
|哲学家|筷子|死锁|chopstick|

# 最后记忆口诀

```
生产消费：
先看有没有位置，有没有产品

多生产消费：
多个生产消费抢资源

抽烟：
等自己的材料

读写：
读可以一起，写必须独占

哲学家：
拿筷子，小心死锁
```

考试做PV题时，基本就是先判断属于哪一个模型，然后套对应模板。你之前学信号量的话，这五个就是最重要的五个“母题”。
